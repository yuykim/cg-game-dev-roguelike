# Rewind Runner 기획서 (확정본)

## 기본 정보

- 프로젝트명: `Rewind Runner` 가제
- 장르: 2.5D 시간조작 액션 플랫포머
- 플랫폼: WebGL 브라우저
- 그래픽 방향: 2D 픽셀 캐릭터 + 3D 도형 맵
- 핵심 목표: 다양한 액션으로 장애물 맵을 돌파하고, 실패 직전 시간을 되감아 최대한 빠르게 클리어하는 게임

## 1. 게임 콘셉트

전투 없는 마리오에 시간 되감기를 더한 스피디한 액션 플랫포머다.

플레이어는 이동, 점프, 대시(구르기), 벽점프를 활용해 장애물을 돌파한다.
낙사하거나 HP가 0이 되면 런이 즉시 종료되기 때문에, 되감기로 위험 직전 상황을 복구하는 것이 핵심 생존 수단이다.

## 2. 게임 흐름

### 전체 구조

```
연습 맵 → 시작 맵 → 랜덤 10개 맵 → 최종 맵
```

### 단계별 설명

1. **연습 맵**: 자유롭게 이동, 점프, 대시, 벽점프, 되감기를 익히는 공간. 제한 없음.
2. **시작 맵**: 이름 입력 → START 버튼 → 3·2·1 카운트다운 → 타이머 시작
3. **랜덤 10개 맵**: 15개 맵 중 랜덤으로 선택된 10개를 순서대로 돌파. 난이도도 랜덤.
4. **최종 맵**: 목적지 도달 → 전체 클리어 타임 표시 → 1~5등 기록 확인

### 사망 처리

- HP 0 또는 낙사 → 런 즉시 종료 → 처음부터 재시작 여부 선택
- 런 전체 기록은 삭제
- 단, 해당 런에서 세운 **맵별 기록이 순위권(1~3등)이라면 저장**

## 3. 플레이어

### HP 시스템

- HP 최대 3
- 위험물 접촉, 레이저 피격 → HP 1 감소
- 낙사 → 즉시 런 종료 (HP 관계없음)
- HP 0 → 런 종료
- **되감기로 피격 전으로 돌아가도 HP는 회복되지 않음**
- 히트박스는 시각적 크기보다 작게 설정

### HP 회복

- 15개 맵 중 3개에 회복 아이템 배치
- 해당 맵 클리어 시 또는 아이템 접촉 시 HP +1 (최대 초과 없음)

### 액션

| 액션 | 설명 |
|------|------|
| 좌우 이동 | 기본 이동 |
| 점프 | 기본 점프 |
| 대시 / 구르기 | **동일한 액션**. 지상: 수평 대시(무적 프레임). 공중: 1회 사용 가능. |
| 벽점프 | 벽에 닿은 상태에서 반대 방향으로 점프. 연속 횟수 제한 없음. |

### 벽 미끄러짐

- 벽에 붙으면 천천히 아래로 내려옴

### 삭제된 액션

- ~~공격~~: 삭제. 오브젝트 조작 기믹 없음.
- ~~웅크리기~~: 삭제. 대시로 대체.
- ~~슬로우모션~~: 삭제.

## 4. 시간 되감기

- 특정 키를 누르고 있는 동안 시간이 역재생됨
- 게이지 **무제한**
- 되감기 버튼을 떼는 순간 해당 위치에서 즉시 조작 가능
- **타이머는 실제 경과 시간만 측정. 되감기로 돌아가도 타이머는 되돌아가지 않음**
- 낙사 직전에 되감기로 복구하는 것이 핵심 생존 플레이

### 되감기 저장 대상

- 플레이어 위치 / 속도 / 애니메이션 상태
- HP (되감기로 회복 불가, 저장은 하되 과거 HP로 복원하지 않음)
- 주요 장애물 위치
- 움직이는 플랫폼 상태
- 카메라 위치

## 5. 장애물

모두 3D 기본 도형으로 구성. 오브젝트 조작 기믹 없음.

| 장애물 | 설명 |
|--------|------|
| 사각형 플랫폼 | 기본 발판, 벽, 이동 플랫폼 |
| 빨간 삼각기둥 | 가시. HP 1 감소. |
| 구체 | 굴러오는 장애물, 낙하물 |
| 회전 큐브 | 타이밍 장애물 |
| 레이저 | 주기적으로 켜지고 꺼지는 위험 구간. HP 1 감소. |
| 구멍 / 낙사 구간 | 닿으면 즉시 런 종료 |

## 6. 맵 구성

### 좌표 구조

- X축: 좌우 이동
- Y축: 점프와 낙하
- Z축: 3D 깊이감 (시각 전용, 충돌은 2D 기준)

### 카메라

- 플레이어 추적, 약간 지연을 두어 부드럽게 따라옴
- 챔버 크기: 화면 기준 1.5배 ~ 3배

### 맵 종류 (총 17개)

- 연습 맵 1개: 튜토리얼
- 시작 맵 1개: 이름 입력, 카운트다운
- 플레이 맵 15개: 난이도 다양. 이 중 3개는 HP 회복 아이템 포함.
- 최종 맵 1개: 클리어 확인

## 7. 기록 시스템

### 전체 기록 (런 기록)

- 10개 맵 전체 합산 클리어 시간
- 1~5등까지 저장

### 맵별 기록

- 각 맵의 개인 최고 기록 1~3등 저장
- 런 실패 시에도 해당 런의 맵별 기록이 순위권이라면 저장됨

### 저장 방식

- localStorage (브라우저 로컬 저장)

## 8. 조작 (키보드 기준)

| 키 | 액션 |
|----|------|
| `A / D` 또는 `←/→` | 좌우 이동 |
| `Space` | 점프 |
| `Shift` | 대시 / 구르기 (지상·공중 동일 키) |
| `Q` | 되감기 |
| `R` | 즉시 재시작 (런 포기) |

## 9. 비주얼 방향

### 주인공

- 2D 픽셀 스프라이트
- Billboard / Sprite 방식 렌더링 (카메라를 향하는 평면)
- 애니메이션: 상태 머신 제어

### 맵

- 3D 박스, 구체, 삼각기둥 중심
- 배경: 어두운 테스트 공간 / 추상 시뮬레이션 룸

### 색상 규칙

| 색상 | 의미 |
|------|------|
| 회색/검정 | 일반 지형 |
| 빨강 | 위험 (즉사 아님, HP 감소) |
| 노랑 | 움직이는 장애물 |
| 파랑 | 되감기 관련 연출 |
| 초록 | 출구 / HP 회복 아이템 |

## 12. 개발 우선순위

1. 2D 평면 물리 + 플레이어 컨트롤러 (이동, 점프)
2. 스프라이트 애니메이션 상태 머신
3. 대시 (무적 프레임, 공중 1회 제한)
4. 벽점프 + 벽 미끄러짐
5. 시간 기록 버퍼 구현
6. 되감기 구현
7. 3D 도형 장애물 배치
8. HP + 피격 + 낙사 처리
9. 맵 15개 제작 + 랜덤 선택 시스템
10. 타이머 + 기록 저장 UI

## 13. 핵심 리스크와 해결 방향

| 리스크 | 해결 |
|--------|------|
| 2D 스프라이트와 3D 맵 정렬 어색함 | 게임플레이는 2D 좌표계 고정, 3D는 시각 깊이만 사용 |
| 되감기 상태 저장 복잡성 | 플레이어와 핵심 오브젝트만 기록, 범위 최소화 |
| 낙사 즉시 종료로 인한 긴장감 과부하 | 되감기 무제한으로 밸런스 조정 |
| 반복 도전 동기 부족 | 맵별 개인 기록 + 전체 순위로 리플레이 유도 |

## 14. 한 줄 요약

2D 픽셀 캐릭터가 랜덤으로 선택된 10개의 3D 도형 챔버를 대시, 벽점프, 시간 되감기로 돌파하고 최단 시간 클리어를 겨루는 WebGL 액션 플랫포머.
