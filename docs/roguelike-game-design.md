# Echo Brawler 기획서

> 현행 프로토타입 기준 문서입니다. 이전 러너/되감기 기획은 폐기하고, 2.5D 1:N 액션 로그라이트 방향으로 확정합니다.

## 1. 핵심 방향

- 장르: 2.5D 사이드뷰 액션 로그라이트
- 플랫폼: WebGL / Three.js
- 비주얼: 2D 스프라이트 캐릭터 + 3D 아레나/플랫폼
- 전투: 무기 없음. 주먹, 발차기, 몸기술, 회피 중심
- 목표 감각: 여러 적에게 둘러싸인 상황을 구르기와 광역 콤보로 풀어내는 난전 액션

핵심 재미는 "한 명을 정교하게 때리는 게임"이 아니라 "몰려드는 적 무리를 위치 선정과 타격감으로 정리하는 게임"입니다.

## 2. 현재 프로토타입 구현 상태

- `Arena`가 현재 메인 런타임입니다.
- 3개 웨이브가 순차적으로 등장합니다.
- 적 타입은 `grunt`, `fast`, `tank` 3종입니다.
- 플레이어는 이동, 점프, 벽점프, 구르기, 3타 콤보 공격을 사용합니다.
- 타격 시 히트스톱, 화면 흔들림, 히트 스파크, 넉백이 적용됩니다.
- HUD는 HP, 런 시간, 웨이브, KO 수를 표시합니다.
- 시작 화면, 사망 화면, 클리어 화면의 최소 구조가 들어가 있습니다.
- 1:N 전투가 과도하게 불공정해지지 않도록 프레임당 실제 접촉 공격 슬롯을 제한합니다.

## 3. 조작

| 동작 | 키 | 역할 |
| --- | --- | --- |
| 이동 | A/D, 방향키 | 좌우 이동 |
| 점프 | Space | 기본 점프, 벽점프 포함 |
| 구르기 | Shift, Ctrl, S | 무적 회피 + 위치 재정렬 |
| 공격 | J | 3타 콤보 |
| 재시작 | R | 현재 런 재시작 |

구르기는 기존 대시를 대체합니다. 공격보다 생존과 위치 재정렬이 먼저이며, 적을 모은 뒤 3타 콤보로 밀어내는 흐름을 기본 전투 루프로 둡니다.

## 4. 전투 설계

### 4.1 기본 콤보

| 단계 | 역할 |
| --- | --- |
| 1타 | 빠른 견제, 약한 넉백 |
| 2타 | 콤보 연결, 약간 강한 넉백 |
| 3타 | 마무리, 높은 넉백과 강한 히트스톱 |

현재는 주먹 콤보만 구현되어 있습니다. 다음 단계에서는 발차기, 충격파, 잡기 중 하나를 먼저 추가해야 전투 선택지가 생깁니다.

### 4.2 1:N 전투 규칙

- 동시에 많은 적이 화면에 존재해도 실제로 피해를 줄 수 있는 적 수는 제한합니다.
- 적은 플레이어를 무작정 추적하되, 서로 겹치지 않도록 간단히 분리됩니다.
- 플레이어는 구르기로 포위망을 빠져나가고, 적이 모인 방향으로 콤보를 넣습니다.

## 5. 적 타입

| 타입 | 현재 구현 | 기획 역할 |
| --- | --- | --- |
| Grunt | 보통 체력, 보통 속도 | 기본 물량 |
| Fast | 낮은 체력, 빠른 속도 | 측면 압박 |
| Tank | 높은 체력, 넉백 저항 | 전선 고정, 압박 중심 |

다음 확장 후보는 원거리 적보다 "점프/돌진/방패" 계열이 좋습니다. 원거리 탄막은 현재 게임의 근접 난전 정체성을 흐릴 가능성이 큽니다.

## 6. 방 구조

MVP는 웨이브 아레나 하나로 충분합니다. 이후 로그라이트 구조로 확장할 때는 방 타입을 나눕니다.

| 방 타입 | 클리어 조건 | 목적 |
| --- | --- | --- |
| 전투 방 | 모든 적 처치 | 기본 전투 |
| 생존 방 | 일정 시간 버티기 | 구르기/위치 선정 검증 |
| 추격 방 | 출구까지 도달 | 플랫폼 액션 강화 |
| 엘리트 방 | 강화 적 처치 | 보상 전 긴장감 |
| 휴식 방 | 보상만 선택 | 리듬 조절 |

## 7. 보상 설계

웨이브 또는 방 클리어 후 3개 카드 중 1개를 고르는 구조를 목표로 합니다.

### 우선 구현할 패시브

- 최대 HP +1
- 기본 공격 피해 +1
- 구르기 쿨다운 감소
- 공격 범위 증가
- 처치 시 낮은 확률로 HP 회복

### 우선 구현할 액티브

- 충격파: 주변 적을 밀어내는 광역 기술
- 그라운드 슬램: 공중에서 착지하며 주변 적 피해
- 잡기/던지기: 적 하나를 잡아 다른 적에게 던짐

첫 액티브는 충격파가 가장 적합합니다. 1:N 전투의 포위 해소 역할이 명확하고, 구현 비용도 낮습니다.

## 8. UI/HUD

현재 HUD에는 다음 항목이 필요합니다.

- HP: 색으로 위험도를 표현합니다. 3 이상은 초록, 2는 주황, 1 이하는 빨강.
- 런 시간: 기록 저장의 기준입니다.
- 웨이브: 현재 진행도와 남은 적 수를 표시합니다.
- KO: 이번 런 처치 수를 표시합니다.

추가로 필요한 화면은 보상 선택 화면입니다. 현재 시작/사망/클리어 화면은 최소 구조만 있으므로, 다음 단계에서 카드 UI를 붙이면 됩니다.

## 9. 기록

localStorage에 저장할 기록은 다음으로 제한합니다.

- 최고 도달 웨이브 또는 방 수
- 최고 KO 수
- 최단 클리어 시간
- 최장 생존 시간

초기에는 랭킹보다 "이전 런보다 나아졌는가"를 보여주는 정도면 충분합니다.

## 10. 개발 우선순위

1. 현재 아레나 프로토타입 안정화: 런타임 오류, HUD, 적 공격 슬롯, 클리어/사망 루프
2. 보상 선택 화면: 웨이브 클리어 후 3개 카드 중 1개 선택
3. 패시브 보상 5종 적용
4. 충격파 액티브 추가
5. 방 3종 추가: 전투, 생존, 추격
6. 기록 저장
7. 적 행동 차별화: Fast 측면 이동, Tank 짧은 돌진

## 11. 현재 남은 기획 리스크

- 주먹 콤보만 있으면 전투가 다시 단조로워질 수 있습니다. 보상과 액티브가 들어가기 전까지는 장기 재미가 약합니다.
- 적 AI가 단순 추적뿐이면 물량이 늘어도 플레이 경험이 비슷해집니다. 최소한 타입별 이동 패턴 차이가 필요합니다.
- 로그라이트 보상은 수치만 올리면 금방 지루해집니다. "위치 선정 방식이 바뀌는 보상"을 우선해야 합니다.

## 12. 다음 작업 추천

다음 개발은 보상 선택 화면과 패시브 3종이 가장 효과적입니다. 현재 프로토타입의 가장 큰 공백은 "웨이브를 깬 뒤 플레이 방식이 달라지지 않는다"는 점이기 때문입니다.
