# Time Echo Runner 기획서 (확정본)

## 0. 기본 방향

- 프로젝트명: `Time Echo Runner` 가제
- 장르: 2.5D 딜레이 고스트 액션 퍼즐 플랫포머
- 플랫폼: WebGL 브라우저
- 그래픽 방향: 2D 픽셀 주인공 + 3D 도형 테스트 챔버
- 핵심 판타지: 현재의 나와 2.5초 뒤 따라오는 고스트를 동시에 계산하며 챔버를 돌파한다.

## 1. 한 줄 설명

`2.5초 뒤의 나`와 협력해서 3D 도형 챔버를 돌파하는 2.5D 액션 퍼즐 플랫포머.

## 2. 핵심 시스템: 상시 딜레이 고스트

플레이어의 과거 행동을 일정 시간 뒤 그대로 재생하는 고스트가 항상 존재한다.

- 플레이어는 현재 캐릭터를 직접 조작한다.
- 고스트는 플레이어의 과거 위치와 행동을 딜레이 후 재생한다.
- 고스트는 플레이어와 물리 충돌하지 않는다.
- 고스트는 지형 충돌을 하지 않고 기록된 위치를 그대로 따라간다.
- 고스트는 일부 장치와 상호작용한다.
- 플레이어는 현재 행동과 미래의 고스트 행동을 동시에 고려해야 한다.

### 고스트 딜레이

고스트 딜레이는 **전 레벨 고정 2.5초**다.
플레이테스트를 통해 조정할 수 있으나, 레벨마다 바꾸지 않는다.

### 죽으면 고스트 버퍼 초기화

플레이어가 사망하면 고스트 녹화 버퍼를 전부 초기화한다.
고스트는 스폰 지점부터 다시 시작한다.

이유: 죽고 다시 시작하는 루프가 단순하고 예측 가능해야 한다. 버퍼가 유지되면 이전 죽음의 행동이 남아 혼란을 준다.

### 고스트 비활성화

고스트가 위험물(가시, 레이저)에 닿으면 **1.5초간 비활성화**된다.
비활성화 중에는:
- 위치는 계속 따라간다
- 장치(버튼, 스위치, 파괴벽)와 상호작용하지 않는다
- 흐릿하게 표시된다

### 고스트가 재생하는 정보

- 위치 (X, Y)
- 방향 (좌/우)
- 애니메이션 상태
- 공격 타이밍
- 구르기 타이밍
- 점프 / 벽차기 타이밍

### 고스트가 할 수 있는 것

- 버튼 밟기
- 스위치 타격
- 파괴벽 공격
- 고스트 전용 발판 활성화
- 문 개방 유지

### 고스트가 하지 못하는 것

- 플레이어를 밀기
- 플레이어를 발판처럼 받치기
- 일반 물리 오브젝트와 충돌
- 체력 회복 아이템 획득

## 3. 핵심 재미

### 피지컬

플레이어는 점프, 구르기, 벽차기, 공격 타이밍을 정확히 수행해야 한다.

### 예측

지금 내가 한 행동이 딜레이 후 고스트 행동으로 재생된다.
플레이어는 현재 장애물뿐 아니라 `미래에 고스트가 어디 있을지`를 계산해야 한다.

### 협동 퍼즐

혼자서는 열 수 없는 문, 누르고 있어야 하는 버튼, 동시에 필요한 스위치를 고스트와 함께 해결한다.

## 4. 핵심 설계 원칙: 기다림 금지

> 고스트가 자동으로 문제를 해결해서 현재 플레이어가 기다리기만 하면 되는 구조는 금지한다.

모든 퍼즐은 `현재 플레이어가 뭔가를 동시에 해야` 고스트의 행동이 의미를 갖도록 설계한다.

**좋은 구조:**
- 버튼이 고스트가 밟기 직전에는 닫혀있어서, 고스트 타이밍에 맞춰 플레이어가 정확히 통과해야 한다.
- 고스트가 스위치를 치는 순간 현재 플레이어가 레이저 구간을 구르기로 통과한다.
- 플레이어가 파괴벽 1타 → 딜레이 후 고스트가 2타. 플레이어는 그 사이 다른 루트를 찾아야 한다.

**나쁜 구조:**
- 고스트가 버튼을 밟는 동안 플레이어가 그냥 기다리다 통과.
- 고스트 없이도 점프로 통과 가능한 구조.
- 단순 피격 장애물만 있는 긴 직선 구간.

## 5. 플레이어 액션

### 이동

- 좌우 이동
- 점프 (코요테 타임, 점프 버퍼 포함)
- 벽차기 + 벽 미끄러짐
- 구르기
- 공격

### 구르기

- 지상과 공중 모두 사용 가능
- 공중에서는 1회 제한
- 빠른 수평 이동
- 짧은 무적 프레임
- 좁은 구간 통과, 레이저/가시 타이밍 돌파
- 고스트도 동일하게 구르기 재생

### 공격

전투용이 아니다. 오브젝트 조작 전용.

- 파괴 가능한 벽 파괴
- 스위치 타격
- 특정 장치 작동
- 고스트도 동일하게 공격 재생

### HP

- 최대 3
- 위험물 접촉 시 1 감소
- HP 0 → 사망, 버퍼 초기화, 레벨 처음부터 재시작
- 사망 무적 시간 없음 (구르기 무적으로 대체)

## 6. 장치 설계

### 버튼

밟고 있는 동안만 활성화된다.

- 플레이어와 고스트 모두 작동 가능
- 버튼이 눌린 동안 연결된 문이 열린다
- 떼면 즉시 닫힌다
- 핵심 퍼즐 패턴: 플레이어가 버튼을 밟고 지나감 → 딜레이 후 고스트가 같은 버튼을 다시 밟음 → 그 타이밍에 현재 플레이어가 문을 통과

### 고스트 전용 버튼

고스트만 작동 가능하다.

- 플레이어가 직접 밟아도 반응하지 않는다
- 고스트 활용을 강제하는 퍼즐에 사용

### 고스트 전용 발판

고스트가 올라서 있는 동안만 활성화되는 발판.

- 플레이어는 올라설 수 없다
- 고스트가 지나간 루트 위에 발판이 생겨 현재 플레이어가 이후에 올라탈 수 있다

### 문

버튼 또는 스위치와 연결된다.

- 닫힌 문은 통과 불가
- 연결된 버튼을 누르는 동안만 열린다
- 떼면 즉시 닫힌다

### 파괴벽

- 일반 파괴벽: 1회 타격으로 파괴
- 강화 파괴벽: 2회 타격 필요 (예: 플레이어 1타 → 딜레이 후 고스트 2타)
- 플레이어와 고스트 모두 타격 가능

### 레이저

- 주기적으로 켜지고 꺼지는 on/off 장애물
- 플레이어에게 피해
- 고스트가 레이저에 닿으면 1.5초 비활성화
- 스위치로 잠깐 끌 수 있다

### 이동 플랫폼

- 기본적으로 왕복 이동
- 고스트가 버튼을 눌러 이동 위치를 바꾸는 플랫폼도 가능

## 7. Echo Window UI

고스트가 버튼을 밟는 순간부터 떼는 순간까지 화면 가장자리에 카운트다운을 표시한다.

예: `ECHO OPENS IN 1.2s` → `ECHO ACTIVE` → `ECHO CLOSES IN 0.8s`

플레이어가 언제 통과할지 판단 기준을 제공한다.

## 9. 랭크와 기록

| 항목 | 기준 |
|------|------|
| S | 빠른 시간 + 피격 0 |
| A | 빠른 시간 |
| B | 클리어 |
| C | 피격 많음 |

저장: 레벨별 최고 시간 + 전체 런 최고 시간 (localStorage)

## 10. 맵 구성

### 튜토리얼 맵

한 맵에서 모든 핵심 개념을 순서대로 학습한다.

1. 이동, 점프
2. 구르기 (좁은 통로)
3. 벽차기
4. 버튼 + 고스트 타이밍 (가장 기본 패턴)
5. 공격 + 파괴벽

### 본 게임 맵 (MVP 5개)

각 맵은 최소 하나의 `고스트 없이는 해결하기 어려운 구조`를 포함한다.

| 맵 | 핵심 메카닉 |
|----|------------|
| 1 | 버튼 + 문 기본 패턴 |
| 2 | 강화 파괴벽 (2타) |
| 3 | 고스트 전용 버튼 |
| 4 | 레이저 + 고스트 비활성화 활용 |
| 5 | 복합 퍼즐 |

## 11. 비주얼 방향

### 플레이어

- 2D 픽셀 스프라이트, 원본 색상

### 고스트

- 같은 스프라이트
- 청록/파랑 틴트 + 반투명
- 플레이어보다 약간 뒤 Z에 렌더링
- 0.5초 잔상 궤적 표시
- 비활성화 시 흐릿하게 표시

### 색상 규칙

| 색상 | 의미 |
|------|------|
| 회색 | 일반 지형 |
| 빨강 | 위험물 |
| 파랑 | 이동 플랫폼 |
| 청록 | 고스트 전용 장치 |
| 주황 | 파괴벽 |
| 초록 | 출구 |

## 12. MVP 범위

### 포함

- 상시 딜레이 고스트 (버퍼 + 재생)

- 고스트 비활성화 (1.5초)
- 고스트의 버튼 / 스위치 / 파괴벽 상호작용

- 고스트 전용 발판
- 구르기 (무적 프레임, 공중 1회)
- 점프 / 벽차기
- 공격 + 파괴벽 (1타 / 2타)

- on/off 레이저
